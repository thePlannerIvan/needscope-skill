# Step 2: 抽样与编码策略设计（v3 — 四类抽样）

## 使用时机

完成 Step 1 范围确认后，在进入 NeedScope 编码之前。**默认不假定全量编码**，根据数据量决定策略。**v3 要求：仅用随机抽样不能声称完成了质量评审。必须包含上下文风险抽样和高互动抽样。**

## 输入

- `work/contracts/00_data_inventory.json`
- `work/contracts/01_scope_decision.json`
- 原始数据文件

## 操作步骤

1. **统计数据量**
   - 每条数据源的记录总数、字段数
   - 去重前和去重后数量
   - 按平台 / 产品线 / 互动量 / 时间分布的记录数

2. **设计四类抽样（v3）**

   正式运行必须至少包含以下抽样类型（按需组合）：

   ### a) 随机抽样（random_sample）
   - 保证统计代表性的基础样本集
   - **所有正式运行必须包含随机样本**
   - 按总量比例随机抽取（如总量的 10%-30%）
   - 用于统计推断和误差估计

   ### b) 高互动抽样（high_like_sample）
   - 高赞/高回复文本的定向抽样
   - 高互动文本可能包含强劲的品牌信号，也可能只是社区/事件热点
   - **必须单独审查**，不得混入随机样本中一并处理
   - 高互动判定标准：如点赞数 ≥100、回复数 ≥50（根据数据分布动态调整）

   ### c) 上下文风险抽样（context_risk_sample）
   - 必须包含以下关键字的文本（或其语义等价表达）：
     - `doge` / `哈哈` / `笑` / `偷笑` / `笑哭` / `评论区` / `弹幕` / `粉丝`
     - 名人/代言人相关表述
     - 活动机制相关表述（限时、联名、打卡）
     - 平台标识或活动标识
   - 用于验证平台/社区/活动信号是否被误归为品牌信号
   - 抽样大小应确保覆盖 Step 0 中识别的所有 `context_risk_flags`

   ### d) 主落点候选抽样（primary_candidate_sample，可选）
   - 可能直接支撑品牌主落点的高质量文本
   - 筛选条件：明确提及品牌评价、有情感倾向的文本
   - 可在 Step 4 编码后根据分布补充

3. **选择基础抽样模式**
   - **< 500 条**：全量编码（如用户预算允许）
   - **500 – 5,000 条**：分层抽样 + 高价值文本全量编码
     - 分层维度：平台、产品线（如可拆分）、互动量（高/中/低）
     - 每个分层至少覆盖 30% 样本
   - **5,000 – 50,000 条**：去重 → 轻量聚类 / 主题分桶后抽样编码
     - 先做文本相似度去重
     - 主题分桶后每个桶抽样 50-100 条
     - 高互动 / 高争议 / 情绪极值文本全量编码
   - **> 50,000 条**：统计预处理 + 主动学习式抽样 + 局部 LLM 复核
     - 先用关键词和正则做初步信号筛选
     - 分桶后每个桶做局部抽样 LLM 编码
     - 用统计方法估计整体分布

4. **记录排除记录**
   - 去重后移除的记录数
   - 抽样未覆盖的记录数
   - 排除理由

5. **记录 rationale**
   - 为什么选择这个抽样策略
   - 抽样误差对结论置信度的影响评估

## 约束

- **不允许只抽热门评论**。必须覆盖低互动评论区。
- **不允许只抽最容易编码的文本**。必须尽量降低抽样偏差。
- **不允许不做去重直接编码**。重复文本会扭曲信号密度。
- **不允许仅用随机抽样就宣称完成了质量评审**（v3）。必须包含上下文风险抽样和高互动抽样。
- 如果用户 `coding_budget_mode` 为 `conservative`，优先用更激进的抽样而非全量编码。
- 如果用户 `coding_budget_mode` 为 `exhaustive`，在预算允许时倾向全量编码。

## 输出文件

- `work/02_sampling_plan.md`：抽样策略说明（人可读）
- `work/contracts/02_sampling_plan.json`：合约

## Contract 字段

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `total_records` | integer | ✅ | 原始数据总记录数 |
| `dedupe_strategy` | string | ✅ | 去重方法（exact / fuzzy / none） |
| `sampling_mode` | string | ✅ | full / stratified / clustered / active_learning |
| `strata` | array | 否 | 分层维度列表 |
| `sample_sizes` | object | ✅ | 每层/每桶的抽样数量 |
| `random_sample` | object | ✅ | **v3**: 随机样本 { size, selection_method } |
| `high_like_sample` | object | ✅ | **v3**: 高互动样本 { size, criteria } |
| `context_risk_sample` | object | ✅ | **v3**: 上下文风险样本 { size, included_risk_flags } |
| `primary_candidate_sample` | object | 否 | **v3**: 主落点候选样本 { size, criteria } |
| `full_coding_subsets` | array | 否 | 全量编码的子集说明 |
| `excluded_records` | integer | ✅ | 排除的记录数 |
| `rationale` | string | ✅ | 策略选择理由及误差影响评估 |

## Completion Criterion

- [ ] 包含随机抽样（random_sample）— 所有正式运行必须
- [ ] 包含高互动抽样（high_like_sample）
- [ ] 包含上下文风险抽样（context_risk_sample）
- [ ] 覆盖了 Step 0 中识别的所有 context_risk_flags
- [ ] 数据量 < 500 → 全量编码
- [ ] 500-5,000 → 分层抽样 + 高价值文本全量
- [ ] 5,000-50,000 → 去重/聚类/分桶后抽样
- [ ] > 50,000 → 统计预处理 + 主动学习式抽样
- [ ] 不允许只抽热门评论或最易编码文本
- [ ] 不允许仅用随机抽样就声称质量评审完成（v3）
- [ ] 已考虑平台、互动量、主题桶或产品线等分层维度

## 常见失败

| 失败 | 后果 | 防范 |
|------|------|------|
| LLM 被当成搬运工全量逐句标注 | token 浪费，成本失控 | 根据数据量选择对应策略，不为"严谨"做全量搬运 |
| 抽样策略没覆盖低互动区 | 信号偏向高互动表达 | 分层标注互动量区间 |
| 没做去重直接编码 | 重复文本扭曲原型信号密度 | 编码前总是先做去重 |
| **仅用随机抽样（v3）** | 高互动/高风险语境中的误归因未被发现 | 必须包含 high_like_sample + context_risk_sample |
