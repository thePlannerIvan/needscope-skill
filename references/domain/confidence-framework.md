# 置信度框架

## 整体分析置信度

由数据源组合和数量决定。在 Step 0 确定，贯穿整个分析。

| 整体标记 | 条件 | 报告可用性 |
|---------|------|----------|
| **决策级** 🟢 | 品牌自身评论 ≥100 条，竞品数据覆盖 ≥3 个品牌，数据源含消费者评论 | 结论可直接用于策略制定 |
| **方向级** 🟡 | 品牌自身评论 30-99 条，或数据源混合（有部分消费者数据但覆盖不全） | 方向明确，部分结论建议验证 |
| **探索级（可参考）** 🟠 | 品牌自身评论 <30 条但 ≥5 条，或仅有达人笔记/品牌官方号但数据量大（≥200条） | 可参考品牌表达策略，但不可直接作为消费者洞察 |
| **探索级（数据不足）** 🔴 | 品牌自身评论 <5 条且仅有少量标题数据（<200条），无任何消费者端信号 | 数据基础过薄，不建议做任何决策判断 |

## 局部结论置信度

对每一个结论（原型得分、竞品占位、产品线判定、战略建议）标注置信度。

| 标记 | 含义 | 评论数门槛 | 标题/搜索词门槛 | 能否直接用于决策 |
|------|------|----------|--------------|---------------|
| **🟢 高** | 多源交叉验证，信号清晰一致 | ≥30 条相关评论 | ≥50 条标题或 ≥20 个搜索词提及 | ✅ 可以 |
| **🟡 中** | 方向明确但数据量有限，或多源信号有轻微矛盾 | 5-29 条 | 10-49 条标题或 5-19 个搜索词提及 | ⚠️ 可以定方向，建议补数据 |
| **🔴 低** | 仅有零星线索，或信号源单一且量少 | <5 条 | <10 条标题或 <5 个搜索词提及 | ❌ 仅供参考，不宜直接决策 |

## 使用规则

### 全局规则

1. **每个六原型得分后面必须跟置信度标记**。比如"呵护型 82 🟢"、"身份型 10 🔴"。
2. **每个竞品的原型判定必须带置信度**。有些竞品可能只有搜索词数据（🟡），有些有完整评论（🟢）。
3. **每条产品线的判定必须标置信度**。如果只有一两句评论，就是 🔴，用推测语气表述（"可能偏向..."而非"定位为..."）。
4. **"信号缺失"不等于"得分低"**。如果一个原型在数据中完全没有对应信号，标注为"无信号 ⚪"而非给低分。
5. **分析总结处给出整体置信度**。如果超过 1/3 的结论是 🔴，在报告开头显著位置声明"本报告为探索性分析，数据基础有限，建议补充数据后复核"。

### 数据源置信度衰减

| 数据源 | 最大局部置信度 | 说明 |
|-------|--------------|------|
| 真实消费评论（电商） | 🟢 高 | 最可靠 |
| 搜索行为数据 | 🟡 中 | 需求信号纯净但缺乏情感全貌 |
| 公众讨论（小红书/知乎） | 🟡 中 | 相对真实但有偏差可能 |
| 达人种草评论 | 🟡 中 | 有策划成分 |
| 品牌官方号内容 | 🔴 低 | 仅反映品牌自我表达 |
| 粉丝场（超话/B站官号） | 🔴 低 | 有社区滤镜 |

### 整体置信度衰减规则

- 仅有品牌官方号内容 + 无任何消费者数据 → 最高"方向级"，所有结论附加"基于品牌自我表达层"前缀
- 仅有达人笔记 + 无素人评论 → 最高"方向级"，标注"有策划成分"
- 品牌官方号 + 消费者评论都有 → 可到"决策级"

## 置信度对报告表达的影响

| 置信度 | 表达规范 |
|--------|---------|
| 🟢 高 | 采用确定性表述："品牌的核心落点为..." |
| 🟡 中 | 采用方向性表述："信号指向..."、"数据表明品牌可能..." |
| 🔴 低 | 采用推测性表述："少量信号暗示..."、"这仅是一个可能的假设，需要更多数据验证" |
| ⚪ 无信号 | 明确说"此原型在数据中未找到对应信号" |

## 低置信度数据在报告中

1. **视觉淡化**：低置信度行加 `opacity: 0.55`（HTML 报告）或用 🔴 标记
2. **显著位置声明**：如果超过 1/3 结论为 🔴，在报告开头声明"探索性分析"
3. **不混淆信源**：低置信结论和高置信结论用不同语气表述，不让用户平等对待
