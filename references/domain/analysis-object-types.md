# 分析对象类型

## 为什么需要区分

NeedScope 分析中，不同的"分析对象"决定了：
1. 编码时的 `signal_owner` 归因逻辑（品牌/产品/个人/内容/社区）
2. 报告的结构和结论口径
3. 平台语境的解释方式
4. 消费者信号的等价性判断

**错误的对象类型判定会导致整套分析失准。** 尤其是创始人/IP 类型被误判为品牌时，signal_owner 归因全面错误。

## 对象类型清单

### brand（品牌）

| 属性 | 说明 |
|------|------|
| 典型特征 | 有品牌名、产品线、渠道、定价体系 |
| 信号归因 | 所有评价"品牌"的信号归 signal_owner=brand |
| 报告口径 | 品牌人格定位、品牌表达 vs 消费者感知错位 |
| 典型场景 | 美妆品牌分析、运动品牌诊断、汽车品牌竞品地图 |
| Fallback 规则 | 如无法确定 → 默认视为 brand |

### product（产品）

| 属性 | 说明 |
|------|------|
| 典型特征 | 单个 SKU、型号，有具体属性讨论 |
| 信号归因 | 评价"产品"的信号归 signal_owner=product |
| 报告口径 | 产品层面的原型信号，不直接等同于品牌人格 |
| 典型场景 | 某款跑鞋的消费者评价分析 |

### product_line（产品线）

| 属性 | 说明 |
|------|------|
| 典型特征 | 多个 SKU 组成的系列（如"Racer 系列"、"Pro 系列"） |
| 信号归因 | signal_owner=product_line |
| 报告口径 | 产品线之间的人格分工、冲突和缺口 |
| 典型场景 | 品牌内部不同系列的市场感知分析 |

### founder（创始人）

| 属性 | 说明 |
|------|------|
| 典型特征 | 自然人、有公开形象和言论、常与品牌绑定 |
| 信号归因 | 评价"创始人"的归 signal_owner=founder |
| 特殊处理 | 不能将创始人信号自动等同于品牌信号 |
| 典型场景 | 马斯克/Tim/雷军等创始人的公众认知分析 |
| 注意 | 创始人型品牌（如 Tesla）可能需要 brand + founder 双对象 |

### public_person（公众人物）

| 属性 | 说明 |
|------|------|
| 典型特征 | 有社会影响力的自然人，不一定有品牌 |
| 信号归因 | personal 信号归 signal_owner=founder（或对应身份） |
| 注意 | 与 founder 的区别在于是否同时是某个品牌的创始人 |
| 典型场景 | 公众人物的人设感知分析 |

### content_ip（内容 IP）

| 属性 | 说明 |
|------|------|
| 典型特征 | 内容是核心产品（频道、专栏、播客、自媒体），有明确内容品牌 |
| 信号归因 | 评价内容的信号归 signal_owner=content |
| 特殊处理 | 需要区分"内容本身"和"创作方"的感知差异 |
| 典型场景 | 影视飓风（Tim）、知识博主、UP主频道 |

### campaign（活动/营销战役）

| 属性 | 说明 |
|------|------|
| 典型特征 | 限时活动、联名款、营销 campaign，有开始和结束 |
| 信号归因 | signal_owner=campaign |
| 注意 | 不长期归入品牌原型，除非成为品牌长期资产 |
| 典型场景 | 某次联名活动的消费者反馈 |

### platform_community（平台社区）

| 属性 | 说明 |
|------|------|
| 典型特征 | 平台或社区本身的氛围、文化、用户群体 |
| 信号归因 | signal_owner=community 或 platform |
| 报告口径 | 不作为品牌人格的构成部分 |
| 典型场景 | B站社区文化 VS 站公开讨论 |

## 混合类型处理

当数据中含有多个对象类型时（常见情况），需要：

1. **在 contract 中标注所有涉及的对象类型**
2. **编码时每一条文本明确 signal_owner**
3. **报告结论分对象陈述**，不合并成一个笼统结论

### 常见混合场景

| 场景 | 对象类型 | 处理 |
|------|---------|------|
| 创始人即品牌（如 Tesla/Musk） | brand + founder | 分别编码 signal_owner，报告区分品牌人格和创始人影响 |
| 内容 IP 即品牌（如 影视飓风） | brand + content_ip + founder | 内容信号、创始人信号、品牌信号分开统计 |
| 粉丝即消费者（如 B站苹果用户） | brand + platform_community | 粉丝场数据标注为 community 语境，不直接等同消费者心智 |

## 误判后果

| 实际类型 | 错误判定为 | 后果 |
|---------|----------|------|
| founder/public_person | brand | 创始人个人特质被归为品牌人格；个人争议被误解为品牌损伤 |
| content_ip | brand | 内容平台文化被归为品牌人格；观众对创作的喜爱被误解为品牌崇拜 |
| platform_community | brand | 社区文化被归为品牌资产 |
| campaign | brand | 短期营销感知被归为长期品牌人格 |
