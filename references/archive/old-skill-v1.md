# NeedScope 品牌原型分析 (v1 — archived)

> 归档自 `SKILL.md` 的原始版本。v2 升级为路由式主文件架构。
> 归档时间：2026-07-08
> 
> 保留原因：v1 的完整分析流程仍有参考价值，尤其是置信度框架和 Phase 0 数据识别的经验。
> 不再活跃使用：v1 是单体提示词式流程，v2 将其拆为七步工作流 + 合约 + 领域知识。

---

## 核心原则

1. **数据决定分析层数** — 不是每次都跑全部。拿到数据先分类，再问用户要跑几层。
2. **词库从数据中生长** — 不要用预设的通用词库去套。先读数据的实际用词，再针对性地建立映射。
3. **输出说人话** — 用六种"消费者买的是 X"来表达原型结论，颜色只作为辅助标注，不单独出现。
4. **每个结论必须带置信度** — 数据量大的结论和数据量小的结论，表达方式必须不同。不要让用户把"只有两条评论"的猜测当成确定结论。

## Phase 0：数据识别与分层判定

### 0.1 自动识别

读取用户提供的所有文件。对每一份数据，根据**字段名 + 内容特征**判断类型：

| 识别特征 | 数据类型 | 可支持的分析层 |
|---------|---------|-------------|
| 含"标题/title""笔记内容""正文"等字段 | 笔记/帖子标题与正文 | 品牌自我诊断 |
| 含"评论""comment""回复"等字段，且文本短、口语化 | 消费者评论 | 品牌自我诊断（权重最高） |
| 含竞品品牌名 + 类似字段 | 竞品数据 | 竞品地图 |
| 含"搜索词""上游""下游""搜索指数"等 | 搜索行为数据 | 竞品地图 + 品牌诊断（交叉验证） |
| 可按产品线/系列/SKU 拆分 | 产品线数据 | 产品线分级 |
| 含"达人""KOL""粉丝数""互动量" | 达人/投放数据 | 品牌诊断（辅助） |

### 0.2 告诉用户你能分析什么

识别完成后，用以下结构告知用户：

```
我识别到你的数据包含：
• [数据源A] → 可以分析 [X层]
• [数据源B] → 可以分析 [Y层]
• [数据源C] → 可以分析 [Z层]

基于这些数据，我可以跑以下分析：

第1层：竞品原型地图 — [有/无数据支持]
  回答：消费者在品类里还有哪些未被满足的情感需求？竞品占据了什么位置？

第2层：品牌优势诊断 — [有/无数据支持]
  回答：消费者现在怎么感受这个品牌？最值钱的情感资产是什么？

第3层：产品线原型分级 — [有/无数据支持]
  回答：不同产品分别打中了什么情感需求？哪几条线在互相打架？

你想跑哪几层？
```

**不要跳过这一步直接分析。** 用户可能只想跑其中两层。但如果用户明确说了"全跑"或数据充足且意图明确，可以直接进入分析。

### 0.3 数据源质量判定（必须做）

数据识别完成后，**必须判断每份数据回答的是什么层面的问题**。不同数据源回答的问题本质不同，不能混为一谈：

| 数据源 | 回答的问题 | 信号性质 | 分析可信度 |
|-------|----------|---------|----------|
| **消费者评论** | "消费者实际怎么感受品牌？" | 真实感受，未过滤 | ⭐⭐⭐⭐⭐ 最高 |
| **消费者搜索词** | "消费者主动想要什么？" | 需求信号 | ⭐⭐⭐⭐ 高 |
| **达人/种草笔记** | "KOL 帮品牌说了什么？" | 有策划成分，半真实 | ⭐⭐⭐ 中 |
| **品牌官方号内容** | "品牌想被怎么感知？" | 品牌自我表达 | ⭐⭐ 低（仅反映意图） |
| **竞品数据** | 取决于竞品数据的类型 | 同上表对应行 | — |

**必须在告知用户可分析层级的同时，明确说明数据源的信号性质：**

```
⚠️ 数据质量提示：
你的数据主要是 [品牌官方号内容]，这回答的是"品牌想被怎么感知"而非"消费者实际怎么感受"。
如果你的目标是了解品牌自我表达策略是否一致 → 当前数据够用。
如果你的目标是了解消费者真实情感 → 建议补充消费者评论数据。

[如果混合数据源] 不同数据源回答不同问题，分析时会分开标注，避免混淆。
```

**数据源标注影响整体置信度判定**：
- 仅有品牌官方号内容 + 无任何消费者数据 → 最高只能到"方向级"，且所有结论自动附加"基于品牌自我表达层"前缀
- 仅有达人笔记 + 无素人评论 → 最高"方向级"，标注"有策划成分"
- 品牌官方号 + 消费者评论都有 → 可到"决策级"

## Phase 1：品类校准与词库构建

### 1.1 扫数据，提取高频情感词

从所有非结构化文本（标题 + 评论 + 搜索词）中：

1. 提取 Top 80-120 个高频形容词、动词、名词（排除纯功能词如"鞋子""跑步"）
2. 按出现频次和情感浓度排序
3. 标注每个词的潜在情感方向

如果数据量 < 100 条，可以直接靠 LLM 阅读理解进行判断；如果 > 500 条，可以用 Python 快速跑词频。

### 1.2 判断品类域

根据高频词的模式，判断品类：

- 高频出现"脚感/透气/支撑/竞速/缓震/配速/PB" → 运动鞋
- 高频出现"成分/肤感/温和/过敏/功效/屏障" → 护肤品
- 高频出现"操控/动力/空间/油耗/内饰" → 汽车
- ...以此类推

如果品类特征不明显，从数据内容（产品名、讨论场景）推断。

### 1.3 构建专属词库

基于品类域 + 数据实际用词，为本次分析建立六原型映射表。

**六个原型的中文表达（必须使用以下标签，不要只用颜色）：**

| 原型标签 | 一句话定义 | 信号词方向（根据数据中的实际用词填入） |
|---------|----------|----------------------------------|
| **突破型**（Red） | 消费者买的是"赢"——对抗、征服、打破常规的快感 | 从数据中提取 |
| **身份型**（Purple） | 消费者买的是"我比你高一档"——地位、稀缺、掌控感 | 从数据中提取 |
| **掌控型**（Blue） | 消费者买的是"我做了最聪明的选择"——理性、专业、透明 | 从数据中提取 |
| **呵护型**（Brown） | 消费者买的是"对自己好一点"——舒适、自然、被照顾 | 从数据中提取 |
| **归属型**（Orange） | 消费者买的是"不出错、和大家一样"——日常、百搭、实用 | 从数据中提取 |
| **愉悦型**（Yellow） | 消费者买的是"当下的快乐"——有趣、自由、松弛、好看 | 从数据中提取 |

词库构建原则：
- 每个原型的信号词从实际数据中摘取，**不要硬套通用词典**
- 同一个词在不同品类可能指向不同原型（"科技"在运动鞋 → 掌控型，在汽车 → 身份型）
- 如果某个原型在数据中几乎找不到对应词，**如实标注为"信号缺失"**，不要强行分配

### 1.4 向用户展示校准结果（可选但推荐）

如果品类特征复杂或数据来源多样，简要展示词库样本。

## 置信度框架（贯穿所有分析层）

### 为什么必须有置信度

同一个分析层内，不同结论的数据基础可能天差地别。比如产品线 A 有 200 条评论支撑，产品线 B 只有 3 条。如果报告里两者用同样确定的语气呈现，用户会把薄数据的猜测当成厚数据的结论，做出错误决策。

### 三级置信度

对**每一个结论**（原型得分、竞品占位、产品线判定、战略建议），标注置信度：

| 标记 | 含义 | 评论数门槛 | 标题/搜索词门槛 | 能否直接用于决策 |
|------|------|----------|--------------|---------------|
| **🟢 高** | 多源交叉验证，信号清晰一致 | ≥30 条相关评论 | ≥50 条标题或 ≥20 个搜索词提及 | ✅ 可以 |
| **🟡 中** | 方向明确但数据量有限，或多源信号有轻微矛盾 | 5-29 条 | 10-49 条标题或 5-19 个搜索词提及 | ⚠️ 可以定方向，建议补数据 |
| **🔴 低** | 仅有零星线索，或信号源单一且量少 | <5 条 | <10 条标题或 <5 个搜索词提及 | ❌ 仅供参考，不宜直接决策 |

### 使用规则

1. **每一个六原型得分后面必须跟置信度标记**。比如"呵护型 82 🟢"、"身份型 10 🔴"。
2. **每个竞品的原型判定必须带置信度**。有些竞品可能只有搜索词数据（🟡），有些有完整评论（🟢）。
3. **每条产品线的判定必须标置信度**。这是置信度出问题最多的地方——如果只有一两句评论，就是 🔴，用推测语气表述（"可能偏向..."而非"定位为..."）。
4. **"信号缺失"不等于"得分低"**。如果一个原型在数据中完全没有对应信号，标注为"无信号 ⚪"而非给低分。
5. **分析总结处给出整体置信度**。如果超过 1/3 的结论是 🔴，在报告开头显著位置声明"本报告为探索性分析，数据基础有限，建议补充数据后复核"。

### 整体分析置信度判定

| 整体标记 | 条件 | 报告可用性 |
|---------|------|----------|
| **决策级** 🟢 | 品牌自身评论 ≥100 条，竞品数据覆盖 ≥3 个品牌，数据源含消费者评论 | 结论可直接用于策略制定 |
| **方向级** 🟡 | 品牌自身评论 30-99 条，或数据源混合（有部分消费者数据但覆盖不全） | 方向明确，部分结论建议验证 |
| **探索级（有参考价值）** 🟠 | 品牌自身评论 <30 条但 ≥5 条，或仅有达人笔记/品牌官方号但数据量大（≥200条） | 可参考品牌表达策略，但不可直接作为消费者洞察使用 |
| **探索级（数据不足）** 🔴 | 品牌自身评论 <5 条且仅有少量标题数据（<200条），无任何消费者端信号 | 数据基础过薄，不建议做任何决策判断 |

关键区别：🟠 和 🔴 都是探索级，但前者至少有足够的数据量可以分析"品牌在说什么"，后者连这个都做不到。报告中必须明确标注当前是哪个等级。

## Phase 2：分层分析

### Layer A — 竞品原型地图（需要竞品数据或搜索词数据）

**目标**：画出品类中主要品牌的 NeedScope 占位，找到情感空白象限。

**方法**：
1. 对每个竞品，用 Phase 1 建立的词库对其标题/评论/搜索词编码
2. 计算每个竞品在六个原型上的信号强度（0-100 相对值）
3. 判定每个竞品的主原型和次原型，**附置信度**
4. 绘制品类 NeedScope 地图

**地图框架**（使用四象限 + 六原型位置）：

```
               活跃/外向 (释放能量)

       愉悦型         突破型
      (快乐/自由)    (对抗/征服)

融合/归属 ←——————————|——————————→ 独立/差异

       归属型         掌控型
      (日常/实用)    (理性/专业)

       呵护型         身份型
      (舒适/自然)    (地位/稀缺)

               内敛/内向 (保存能量)
```

**空白识别逻辑**：
- 象限无人占据 → 战略空白（最难，需教育市场）
- 有需求信号但无强势品牌 → 抢占空白（机会最大）
- 本品牌已有信号但被压制 → 需要巩固防守

**输出**：
- 品类 NeedScope 地图（文本版 + 数据）
- 每个竞品的一句话原型判定，附置信度
- 若某竞品仅有搜索词数据而无评论数据，标注为 🟡 并说明缺少消费者原声验证
- 空白象限与机会象限

### Layer B — 品牌优势诊断（需要品牌自身笔记 + 评论数据）

**目标**：找出品牌当前最强的 NeedScope 资产，识别"品牌漂移"。

**方法**：
1. 对品牌标题和评论分别编码六原型得分
2. 评论数据的权重设为标题的 3 倍（评论是未过滤的消费者真实感受）
3. 如果有关键词搜索词数据，作为辅助参考

**权重体系**：
| 信号来源 | 权重 | 理由 |
|---------|------|------|
| 消费者评论（品牌笔记下） | ×3 | 最接近真实感受，未过滤 |
| 消费者主动搜索词 | ×2 | 暴露未被满足的需求 |
| 达人/品牌标题 | ×1 | 有策划成分，反映品牌意图而非消费者感受 |
| 竞品评论中提及本品牌 | ×4 | 跨品牌比较中的提及最有诊断价值 |

**诊断输出**：
- 六原型雷达数据（得分 + 排名 + 置信度 + 数据量）
- **核心资产**：得分最高的 1-2 个原型，附代表性消费者原声引用
- **薄弱环节**：得分最低的 1-2 个原型，区分"得分低但有信号"和"完全无信号"
- **品牌漂移判断**：品牌自我表达（标题）和消费者感受（评论）之间是否有落差？
- **矛盾/张力信号**：消费者评论中出现的两极分化表达（如"丑但舒服"），这往往是品牌最独特的张力所在

### Layer C — 产品线原型分级（需要可按产品线拆分的数据）

**目标**：理解不同产品线分别满足什么 NeedState，发现产品线之间的冲突或空白。

**方法**：
1. 按产品线/系列/SKU 对数据分组
2. 每组独立跑六原型编码
3. 绘制产品线 × 原型矩阵

**输出**：
- 每个产品线的主原型 + 次原型，**强制附带数据量和置信度**
- **数据量不足的产品线（<5条评论）必须明确标注为 🔴 低置信度**，用推测语气表述
- 产品线冲突检测：哪些产品线在争同一个原型？
- 原型覆盖缺口：哪个原型没有产品线承接？
- 代表性产品线评论引用（仅高/中置信度产品线提供）

## Phase 3：输出

### 3.1 Markdown 报告

生成一份结构化的 Markdown 报告。报告遵循这个模板：

```
# [品牌名] NeedScope 品牌原型分析

> **整体置信度：[决策级/方向级/探索级]**
> 数据基础：品牌评论 X 条，笔记标题 X 条，竞品覆盖 X 个品牌
> [如果探索级] ⚠️ 本报告数据基础有限，结论仅供方向参考

## 一、品类全景
[如果跑了 Layer A]
- 品类 NeedScope 地图描述
- 竞品原型占位一览（每个竞品附置信度）
- 空白象限与机会

## 二、品牌自我诊断
[如果跑了 Layer B]
- 六原型得分总览（每个得分附置信度和数据量）
- 核心情感资产（附消费者原声）
- 品牌漂移判断
- 矛盾/张力信号

## 三、产品线分级
[如果跑了 Layer C]
- 产品线 × 原型矩阵（每条线标置信度 🔴🟡🟢）
- 冲突线与空白线
- [显著位置] 低置信度产品线汇总：哪些产品线数据不足，需要补充哪些数据

## 四、战略建议
[综合所有已跑的分析层]
- 每个建议标注置信度（🟢🟡🔴）

## 五、数据补充建议
[如果整体置信度为探索级或方向级，必须包含此章节]
根据当前数据缺口，如果希望升级为决策级报告，建议收集以下数据：
1. [具体数据类型 + 建议数量 + 为什么需要]
2. ...
例如：
  "1. 品牌消费者评论 ≥100 条 — 当前仅靠标题分析，无法验证消费者真实感受"
  "2. 竞品消费者评论 ≥50 条/品牌 — 当前竞品仅有官方号数据，无法对比消费者心智"
```

## 重要提醒

### 关于数据质量

- 如果某个分析层的数据量太少（如评论 < 20 条），**明确告知用户这个结论的置信度有限**
- 如果数据来源单一（如只有笔记标题没有评论），分析结论应标注为"基于内容表达层"
- 高互动笔记（点赞 > 1000）和低互动笔记的信号价值不一样，编码时可考虑互动量加权

### 关于品类重力

每个品类有自己的"情感重心"。运动鞋整体偏外向独立，护肤品整体偏内向归属。这不代表所有品牌都应该往品类重心走——**逆品类重力的品牌往往成为品类定义者**（如 Corona 在啤酒品类的归属型红海中占据愉悦型）。报告中应标注品类重力，但不被它限制判断。

### 关于原型不是"好/坏"

六个原型没有优劣之分。一个品牌在"突破型"得分低不一定是坏事——如果它的策略就是"呵护型"，那突破型得分低恰恰说明品牌表达一致。**诊断的目的不是让品牌得满分，而是帮助品牌确认"我到底是谁"和"消费者觉得我是谁"之间的重合度。**
